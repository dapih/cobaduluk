First public release of **excel-to-json** — a token-frugal pipeline that converts complex Excel tables into validated, schema-backed JSON. Python handles every row; the model supervises structure, schema, parser authoring, and data-quality review.

## Highlights

- **Full pipeline:** inspect → map → schema → parse → validate → data-quality → summary, with mandatory gates and row conservation (`rows in == entries out`).
- **Cross-tool install:** Claude Code marketplace (`excel-to-json@cobaduluk`), [skills.sh](https://skills.sh/dapih/cobaduluk/excel-to-json), and `npx skills add` for Cursor, Codex, OpenCode, Kilo, Antigravity, OpenClaw, Hermes, and 60+ agents.
- **Nested install support:** `scripts/resolve_plugin_root.py` finds the plugin when cloned under `tools/excel-to-json/` in a user project.
- **Same-family reuse:** structural fingerprinting, conformance checks, and opt-in canonical warm-start (never skips validation).
- **Verification gates:** `scripts/verify_install.py`, `scripts/validate_marketplace.py`, and `scripts/smoke_test_compat.py`.

## Install

```bash
# Claude Code
/plugin marketplace add dapih/cobaduluk
/plugin install excel-to-json@cobaduluk

# Cursor / Codex / Kilo / OpenCode / Antigravity / OpenClaw / Hermes
npx skills add dapih/cobaduluk --skill excel-to-json --agent cursor -y
git clone https://github.com/dapih/cobaduluk.git tools/excel-to-json
cd tools/excel-to-json && pip install -r requirements.txt && python scripts/link_skill_discovery.py
```

See [INSTALL.md](https://github.com/dapih/cobaduluk/blob/v0.1.0/INSTALL.md) for per-tool commands.

## Requirements

- Python 3.9+
- `pip install -r requirements.txt` (`openpyxl`, `jsonschema`)

- Kilo commands (`.kilo/commands/`) and Antigravity workflow (`.agents/workflows/`)
